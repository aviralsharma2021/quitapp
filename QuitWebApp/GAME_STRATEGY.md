# QuitWebApp Game Strategy (Kwit-inspired, denser milestone cadence)

## Product goal
Create enough meaningful, fast feedback that a quitter wants to open the app multiple times a day, especially after day 7 when many apps feel sparse.

## Core loop (60-120 seconds)
1. Open app.
2. See progress gains since last check (time, money, body recovery).
3. Complete one micro-action (craving rescue, challenge, check-in).
4. Get immediate reward (XP + chance of bonus).
5. Reveal next near-term target (within hours or 1-2 days).

## Psychological architecture
- Self-Determination Theory:
  - Autonomy: user picks quit strategy, challenge types, tone.
  - Competence: visible skill progression (cravings resisted, rescue streak).
  - Relatedness: future-self trust score + letters.
- Contingency management:
  - frequent contingent rewards early for abstinence-supporting behaviors.
- Loss aversion (gentle):
  - streak shield protects one slip; no shame spiral.
- Variable rewards (ethical):
  - one mystery drop window per day, capped to avoid compulsive loops.

## Progression model

### 1) Level ladder (0-100)
- Fast early levels, gradual slowdown.
- Time XP + behavior XP + challenge XP.
- Every level grants one of:
  - cosmetic badge
  - small XP booster (24h)
  - shield fragment

### 2) Four parallel milestone tracks
- Health track (research-backed physiological recovery milestones)
- Money track (savings thresholds)
- Nicotine-freedom track (clean-time thresholds)
- Resilience track (cravings resisted / rescue streak)

This ensures at least one unlock opportunity every 24-72h in early-mid journey.

## Milestone cadence (dense by design)

### Phase A: Acute withdrawal (Day 0-14)
- Daily major milestone: D1, D2, ... D14.
- Plus sub-day milestones in first 72h (20m, 8h, 24h, 48h, 72h).
- Rationale: highest withdrawal pressure and relapse risk.

### Phase B: Consolidation (Day 15-30)
- Milestones every 2 days (D16, D18, D20 ... D30).
- Add weekly boss challenge (7-day behavior consistency).

### Phase C: Identity build (Month 2-3)
- Milestones every 3-4 days, then weekly.
- Emphasis shifts toward identity and skill mastery.

### Phase D: Maintenance (Month 4+)
- Weekly/biweekly major milestones.
- Daily micro-goals and streak economy still active.

## XP economy
- Passive XP (time smoke-free): small and steady.
- Active XP:
  - craving rescue complete: +45 XP
  - log resisted craving: +20 to +50 XP by intensity
  - daily challenge: +25 to +70 XP
  - milestone unlock: +40 to +300 XP
- Anti-inflation: daily XP soft-cap for repetitive actions.

## Craving interruption design (must feel rewarding)
- "Panic button" starts 90-second urge-surf challenge.
- Animated countdown ring + breathing prompt.
- Completion triggers:
  - immediate XP
  - confidence message
  - chance of mystery drop

## Mystery reward design (bounded variable reinforcement)
- Max 1 chest/day.
- Reward table:
  - 60%: +20-40 XP
  - 25%: shield fragment
  - 10%: full shield
  - 5%: rare title token

## Relapse handling
- Language: "slip" not "failure".
- If shield available: consume shield, streak remains.
- If no shield: streak resets, but keeps:
  - total XP
  - unlocked knowledge cards
  - future-self trust can recover quickly with 48h consistency.

## Why this should fix day-8 boredom
- Day 8-30 includes explicit milestone density (every 1-2 days plus micro tracks).
- Each session surfaces "next unlock ETA" measured in hours/days, not weeks.
- Multiple concurrent tracks prevent dead zones.

## Safety + ethics
- App includes optional links to quitline/clinician support.
- Never replaces medical care.
- No dark patterns (infinite loot loops, guilt shaming, paywall pressure).
